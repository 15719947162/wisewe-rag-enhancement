# 页面覆盖：召回管理

> 本文件中的规则仅覆盖 `docs/design/system/MASTER.md` 在召回管理页面中的部分。

## 页面目标

这是一个可解释召回管理工作台。用户需要在这里查看普通 RAG、Graph RAG、召回、重排、上下文组装、答案生成和评分判断的全过程。

## 语言覆盖

- 本页面所有可见 UI 文案必须使用 `简体中文`。
- `top_k`、`min_score`、`kb_id`、`RRF`、`chunk id` 等必要技术字段可保留英文。
- 所有字段标签、说明文案、状态解释、评分解释、引用说明、失败提示必须为中文。
- “无法回答”、低分原因、错误回退等关键状态，不得输出英文默认文案。

## 核心用户任务

- 输入问题和查询参数
- 查看各召回通道结果
- 检查重排后的候选内容
- 查看最终上下文窗口
- 校验答案和引用关系
- 诊断低忠实度或无法回答场景

## 布局覆盖

桌面端使用 `三栏追踪布局`：

1. 左侧：查询输入区与检索控制区
2. 中间：候选列表、重排结果、上下文窗口
3. 右侧：最终答案、引用、评分面板

页面气质要像可阅读的分析工作台，不是缩小版聊天框。

在较小屏幕上，折叠为逐段堆叠面板。

## 必备面板

### 查询面板

- 问题输入
- 知识库选择器
- `top_k`
- `min_score`
- `use_llm_check`
- `use_llm_score`
- 历史问题默认按最近优先保留，不只硬截断为最新 20 条；窄侧栏内每页展示 20 条并使用简易分页。
- 当历史问题需要跨设备、租户级排障或审计追踪时，应转到查询日志 / 后端分页列表承载，不继续扩大左侧窄栏。

### 召回面板

- dense / sparse / structured 候选数量
- RRF 合并结果
- related expansion 标记
- 候选来源分布

### 重排面板

- 子 chunk 分数
- 父级聚合结果
- 最佳子 chunk 标识
- 上下文窗口预览

### 答案面板

- 答案正文
- 无法回答状态
- 映射到来源、页码、chunk id 的引用
- 错误回退状态

### 评分面板

- 相关性评分
- 忠实度评分
- LLM 评分
- 结果解释文案

## 关键设计规则

- 答案必须始终支持按引用回溯证据
- 候选排序必须明确展示分数标签
- 上下文窗口必须像可阅读文档，而不是缩成小提示框
- 低分案例必须明确说明为什么低分
- 回退、拒答和证据缺失都要有显式状态
- 历史问题属于辅助召回入口，不应挤占查询参数和候选证据主任务空间。
