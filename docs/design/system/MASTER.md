# WiseWe RAG Console 设计系统

> 状态：现行标准  
> 生效日期：2026-06-25  
> 唯一视觉基准：`docs/design/previews/wisewe-rag-console-ui-preview.html`

## 1. 设计定位

WiseWe RAG Console 是教育教材知识库与 RAG 治理控制台，不是营销页，也不是早期技术验证壳。界面必须优先回答四件事：

- 系统是否健康
- 任务卡在哪里
- 风险是什么
- 下一步去哪里处理

历史 UI 规范已经迁移到 `docs/design/archive/legacy-ui-spec-2026-06-25/`，不再参与 active 决策。

## 2. 规范优先级

1. `docs/design/previews/wisewe-rag-console-ui-preview.html`

任何与预览冲突的旧文档、旧卡片套路或旧交互标准，全部按 archive 处理。

## 3. 全局骨架

- 画布：浅灰蓝 `#F5F7FB`，允许极轻网格与低透明域色光感。
- 侧栏：`286px` sticky 左侧导航，白底半透明，不做内容卡片。
- 顶栏：`76px` sticky 顶栏，承载面包屑、搜索、创建入口、身份切换和退出。
- 内容区：桌面最大宽度 `1480px`，左右 `30px` 内边距，移动端 `16px`。
- 圆角：面板、卡片、按钮统一 `8px` 左右，不上浮成大圆角。

## 5. 字体与文案

- UI / 标题：`Inter`, `Noto Sans SC`, `Microsoft YaHei`, system sans
- 数字 / ID / 日志：`JetBrains Mono`, `Fira Code`, `Cascadia Code`
- 可见文案以简体中文为主，英文只保留产品名、协议名、字段名和必要技术标识。
- 页面标题要短，正文要克制，避免营销口吻。

## 6. 组件语法

### 6.1 侧栏

- 分组导航保持扁平，不使用组卡片外壳。
- 分组标题前使用小型域色信号点。
- 菜单项使用 24px 彩色图标底。
- 当前项使用左侧 `3px` 域色条、浅 tint 背景和右侧竖向状态胶囊。
- hover 只改变背景、边框、文字色，不缩放、不位移。

### 6.2 卡片与面板

- 卡片是信息单元，不得回到“卡片套卡片”和“每张卡顶部彩条”的旧模式。
- 背景采用白底 + 浅色线性渐变 + 右下柔和径向色块。
- 边框采用低透明业务域色或 `#DDE3F0`。
- 阴影保持轻量，优先表达层级，不做浮夸漂浮感。
- 指标数字统一使用等宽字体，hover 不改变布局。

### 6.3 表格与分页

- 真实业务表格默认分页，每页 `20` 条，可选 `20 / 50 / 100`。
- 分页控件靠右。
- 窄侧栏、卡片内列表或宽度不足的结构允许使用简易分页：隐藏总数说明与“每页/条”等辅助文案，仅保留条数选择、页码和 `<` / `>` 翻页图标。
- 不要为了适配窄结构直接丢弃历史数据；入口区可轻量展示，完整列表应通过分页、查看全部或日志页承载。
- 表头使用浅 tint 背景。
- 行 hover 使用浅 tint，不改变行高。
- ID、requestId、时间戳、数值使用等宽字体。
- 状态列使用 Badge，不只靠文字或颜色。

### 6.4 输入、按钮与状态

- 主按钮、次按钮、幽灵按钮统一 8px 圆角。
- hover 只做颜色、边框、阴影微变化，不做 scale / translate。
- loading、empty、error、disabled 必须有显式状态。
- 焦点状态必须可见，且适合键盘导航。

## 7. 页面模板

### 总览

- 首屏是左侧指挥台 + 右侧健康 / 身份面板。
- 下方展示核心指标、入库生产线、风险告警、最近任务。

### 知识库列表

- 顶部是知识资产首页，不做营销 hero。
- 必须包含创建入口、总数指标、知识库卡片和快捷入口。

### 入库管理

- 作为生产线台账表达，不展示原始 SSE 实时日志作为主体验。
- 阶段耗时、最新日志和运行中遮罩保留，但主界面优先展示业务状态。

### 召回与评测

- 以可解释工作台为主，不把上下文压成小提示框。
- 候选、重排、答案、引用和评分必须同屏可回溯。

### 日志与用量

- 查询日志、入库日志、用户及权限同步、审计日志明确分层。
- Token 统计按链路、环节、模型调用明细展示。

### 配置与 API

- 作为治理控制面，不做装饰性面板。
- 配置项、来源、覆盖状态和保存状态必须清晰。

## 8. 响应式与可用性

- 375px、768px、1024px、1440px 都必须可用。
- 移动端允许折叠侧栏与底部导航，但不能丢失主任务入口。
- `prefers-reduced-motion` 必须尊重。
- 文案不能溢出容器，长 ID 和长状态需换行或截断。

## 9. 禁用项

- 不再使用 `docs/design/ui-interaction-standard.md` 或 `docs/design/product-ui-redesign-plan.md` 作为实现入口。
- 不再使用 `docs/design/ui/**` 作为 active 规范。
- 不使用深色全局后台。
- 不使用大面积单一紫蓝渐变。
- 不使用纯装饰 hero。
- 不在控制台页面堆叠嵌套卡片。
- 不使用 hover 位移、缩放或慢动画制造“活泼感”。

## 10. 实现提醒

- 视觉一致性优先于单页花活。
- 页面差异只允许写入 `docs/design/system/pages/`。
- 任何新组件都要先对齐这个 Master，再写实现。
