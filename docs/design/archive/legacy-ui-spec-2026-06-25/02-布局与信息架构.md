# 02 布局与信息架构

## 1. 整体布局模型

推荐全站采用统一后台壳结构：

- 左侧主导航
- 顶部上下文栏
- 中央主工作区
- 右侧可折叠详情抽屉

## 2. 主导航

### 一级导航建议

- 总览
- 知识库
- 离线建库
- 在线问答
- 评测分析
- 系统配置

### 导航原则

- 一级导航承载“业务域”，不要承载过细的工具项
- 当前页面的二级导航应放在顶部上下文栏或页内 tabs 中
- 不要让用户在左侧导航里理解完整链路，链路应在页面内部表达

## 3. 顶部上下文栏

顶部栏应承载当前工作上下文，例如：

- 当前知识库 `kb_id`
- 当前文档
- 当前任务
- 当前策略
- 最近运行时间
- 全局搜索 / 全局命令入口

## 4. 主工作区

### 4.1 页面宽度

- 主内容推荐最大宽度 `1440px`
- 数据表格可局部撑满
- 阅读型内容区建议控制在更窄的文本行宽内

### 4.2 栅格系统

- 统一 `12 栅格`
- 默认间距：`24px`
- 小卡片与数据列区域可降到 `16px`

## 5. 右侧详情抽屉

适合承载：

- chunk 明细
- citation 明细
- 模型参数
- 错误详情
- trace 详情

原则：

- 详情应辅助主任务，而不是打断主任务
- 抽屉优先于整页跳转

## 6. 信息架构分层

### 6.1 总览层

回答：系统目前怎么样。

### 6.2 工作台层

回答：我要如何完成一次操作或排查。

### 6.3 证据层

回答：这个结果具体依据什么。

### 6.4 配置层

回答：这个链路按什么规则执行。

## 7. 数据密度分区

### 舒适区

用于：

- 总览指标
- 页面头部
- 关键状态卡

### 标准区

用于：

- 常规工作台内容
- 参数区
- 大多数卡片和列表

### 密集区

用于：

- evidence table
- retrieval candidates
- 评测样本列表
- 诊断明细

## 8. 分栏策略

### 离线建库

推荐 `2.5 栏`：

- 左：任务列表
- 中：阶段结果
- 右：详情

### 在线问答

推荐 `3 栏`：

- 左：问题输入与参数
- 中：召回/重排/上下文
- 右：答案/引用/评分

### 评测分析

推荐 `上汇总，下诊断`

- 上：分数卡、趋势图、分类统计
- 下：失败样本列表与详情

## 9. 响应式要求

### Mobile

- 左侧导航收起
- 右侧详情抽屉改为全屏底部面板
- 三栏合并为垂直分步

### Tablet

- 可保留左侧导航
- 右侧详情改为覆盖式抽屉

### Desktop

- 允许完整分析台结构

## 10. 信息优先级规则

页面中信息优先级顺序应遵循：

1. 当前状态
2. 核心结果
3. 关键异常
4. 可执行动作
5. 证据细节
6. 参数与次要元信息
