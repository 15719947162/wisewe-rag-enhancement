# 06 状态、交互与可用性规范

## 1. 状态体系

统一使用以下状态：

- `待执行`
- `进行中`
- `成功`
- `降级`
- `失败`
- `空状态`

## 2. 状态表达规则

每个关键状态都必须同时包含：

- 颜色
- 图标
- 文字

不能只靠颜色表达。

## 3. 加载状态

### 适用方式

- 页面级：skeleton + 占位区块
- 局部数据区：行级 skeleton
- 图表：固定容器骨架

### 不建议

- 全局菊花长时间遮挡

## 4. 错误反馈

### 错误提示必须包含

- 发生在哪个阶段
- 错误级别
- 原因摘要
- 可执行操作

### 无障碍要求

- 使用 `role=alert` 或 `aria-live`

## 5. Hover / Focus / Selected

### Hover

- 仅改变颜色、边框、阴影
- 不允许造成布局跳动

### Focus

- 所有按钮、输入、表格行、tag 选择器都有清晰 focus ring

### Selected

- 列表选中行必须明显，不与 hover 混淆

## 6. 表格交互

所有高密表格必须支持：

- 排序
- 筛选
- 固定表头
- 行点击查看详情
- 批量选择

## 7. 参数交互

### 原则

- 常用参数外露
- 高级参数收纳
- 参数变化要有即时反馈

### 常用参数示例

- strategy
- top_k
- min_score
- clean
- clean_llm
- use_llm_check
- use_llm_score

## 8. 文本与证据交互

- 答案中的 citation 编号要能 hover 与点击
- citation 卡要能回跳到 chunk
- chunk 详情要能定位 source / page

## 9. 可访问性要求

- 对比度不低于 `4.5:1`
- 所有 icon-only 按钮必须有 `aria-label`
- Tab 顺序符合视觉顺序
- 支持键盘完成主要工作流
- 支持 `prefers-reduced-motion`

## 10. 空状态规范

空状态应解释：

- 为什么没有数据
- 下一步应该做什么

例如：

- 尚未上传 PDF
- 当前 KB 没有任何文档
- 当前查询未召回结果

## 11. 失败恢复机制

页面中应为失败场景提供恢复操作，例如：

- 重试
- 查看详情
- 更换参数
- 降级执行

## 12. 文案规范

- “降级”优于“部分成功”
- “无法回答”优于“无结果”
- “质量过滤丢弃”优于“未通过”
- “证据来源”优于“引用信息”
