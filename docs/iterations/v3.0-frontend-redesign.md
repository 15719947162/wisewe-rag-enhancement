## 背景与目标

v2.0 完成了完整的技术链路验证（云端解析 → 向量存储 → RAG 问答 → 服务端规范化），所有后端能力已就绪。

v3.0 的目标是将这些技术成果转化为**对外可用的产品级前端**，不再是内部调试控制台，而是一个完整的 AI 知识库平台。

**核心变化：**
- 路由分组从 `(console)` 改为 `(rag-api)`
- 所有页面从静态 mock 数据改为接真实后端 API
- 新增上传、创建、实时进度等核心交互
- 后端补充 SSE 推送、知识库 CRUD、评测等缺失 API

## 产品定位

**产品名：** WiseWe 知识库平台  
**核心价值：** 把任意 PDF 文档变成可问答的 AI 知识库，全链路透明可追溯  
**目标用户：** 教育机构、企业内容团队、研究人员  
**实时推送方案：** SSE（Server-Sent Events）— 单向推送，FastAPI 原生支持，比 WebSocket 轻

## 详细功能规格

### 模块一：知识库管理（`/knowledge-bases`）

#### 1.1 知识库列表视图
- 卡片式列表，每张卡片展示：名称、描述、文档数、切块总数、最近更新时间、状态徽标
- 支持按名称搜索、按状态筛选（全部 / 正常 / 建库中 / 有错误）
- 空状态：引导用户创建第一个知识库，说明下一步操作

#### 1.2 创建知识库
- 触发：右上角「新建知识库」按钮
- 表单字段：名称（必填）、描述（选填）、默认切片策略（下拉，6种，带策略说明 tooltip）
- 提交后自动生成 `kb_id`（slug 格式），展示在卡片上
- 创建成功 → Toast 提示 → 列表刷新

#### 1.3 知识库详情页（`/knowledge-bases/[kb_id]`）
- 顶部：知识库名称、`kb_id`、描述、统计数字（文档数、切块数、向量数）
- 主区域：文档列表表格
  - 列：文件名、状态、切片策略、切块数、文件哈希（前8位）、入库时间、操作
  - 支持按状态筛选、按时间排序
  - 行点击 → 右侧抽屉展开文档详情

#### 1.4 文档详情抽屉
- 文档基本信息：文件名、大小、页数、文件哈希
- 建库参数：使用的切片策略、清洗配置
- 切块预览：分页展示切块列表，每条显示 layer（parent/child/enhanced）、内容摘要、页码
- 操作：重新建库（选择新策略）、删除文档

#### 1.5 删除知识库
- 二次确认弹窗，展示将删除的文档数和切块数
- 删除后重定向到列表页

**验收标准：**
- AC-KB-01：可创建知识库，`kb_id` 自动生成且唯一
- AC-KB-02：知识库列表支持搜索和状态筛选
- AC-KB-03：文档列表支持排序，行点击打开抽屉
- AC-KB-04：删除前有二次确认，删除后数据清除

### 模块二：文档入库（`/ingestion`）

这是整个产品最核心的操作页，完整展示 10 个阶段的实时进度。

#### 2.1 上传入口
- 拖拽上传区 + 点击选择，支持 PDF 格式，最大 100MB
- 上传前配置：
  - 目标知识库（下拉选择，可新建）
  - 切片策略（6种，带策略说明 tooltip）
  - 是否启用 LLM 清洗（开关，标注会增加耗时和费用）
- 支持批量上传（多文件排队）

#### 2.2 任务队列（左侧栏）
- 实时展示所有任务：文件名、目标知识库、当前阶段、状态徽标、开始时间
- 点击任务 → 主区域切换到该任务详情
- 状态颜色：待处理（灰）/ 进行中（蓝动画）/ 成功（绿）/ 降级（橙）/ 失败（红）

#### 2.3 阶段步进器（主区域顶部）

10 个阶段横向展示，每个阶段节点包含：`阶段名 → 状态图标 → 输入数 → 输出数 → 耗时`

| 序号 | 阶段名 | 说明 |
|------|--------|------|
| 1 | 文档接入 | 文件校验、格式检查 |
| 2 | OSS 上传 | 进度百分比 |
| 3 | 云端解析 | MinerU，轮询状态 |
| 4 | 内容规范化 | ContentBlock 生成 |
| 5 | 内容清洗 | 规则 + 可选 LLM |
| 6 | 切片 | 策略执行 |
| 7 | 关联补链 | related_ids 构建 |
| 8 | 质量过滤 | 门控评分 |
| 9 | 向量化 | Embedding 生成 |
| 10 | 索引落库 | CSV + pgvector |

#### 2.4 各阶段详情视图

**解析阶段视图**
- PDF 元数据：页数、文件大小、解析方法
- ContentBlock 类型分布：文本块 N 个、表格块 N 个、图片块 N 个（横条图）
- 内容块预览列表：类型标签 + 页码 + 内容摘要（前100字）
- 支持按类型筛选

**清洗阶段视图**
- 左右对比面板：
  - 左：清洗前 block 列表（带类型标签）
  - 右：被删除的 block 及删除原因（空块/过短/纯标点/广告/重复图片）
- 删除原因分布横条图
- LLM 清洗决策（若启用）：展示每条 LLM 判断理由

**切片阶段视图**
- 策略选择器（可在此切换策略重新切片，用于对比）
- 切块统计：总数、parent 数、child 数、enhanced 数
- 切块树结构预览（hierarchical 策略时展示三层缩进树）
- 切块类型分布：文本/表格/图片/增强
- 切块列表：支持按 layer 筛选，点击展开全文

**质量过滤视图**
- 通过数 vs 丢弃数（大数字对比展示）
- 丢弃原因分布：标点比例过高 N 个、LLM 评分过低 N 个
- 评分分布直方图（0-5分区间）
- 被丢弃的切块列表（可展开查看内容和丢弃原因）

**向量化 & 落库视图**
- Embedding 生成进度（批次进度条）
- 模型名称、向量维度
- CSV 导出路径和文件大小
- pgvector 写入结果：写入数、跳过数（重复）、知识库 ID

#### 2.5 SSE 实时推送
- 后端新增 `GET /api/ingestion/stream/{task_id}` SSE 端点
- 前端订阅后实时更新阶段状态、进度数字、日志
- 连接断开自动重连（最多3次），降级为5s轮询

#### 2.6 错误处理
- 每个阶段失败时：展示错误类型、错误信息、建议操作
- 支持从失败阶段重试（不重跑已完成阶段）
- 降级状态（如 LLM 清洗失败回退到规则清洗）明确标注

**验收标准：**
- AC-ING-01：可上传 PDF，选择知识库和策略后启动建库
- AC-ING-02：10个阶段实时更新，每阶段展示状态/输入数/输出数/耗时
- AC-ING-03：清洗阶段展示前后对比
- AC-ING-04：切片阶段可切换策略重新切片
- AC-ING-05：失败阶段展示错误原因，支持重试
- AC-ING-06：SSE 断线自动重连，降级为轮询

### 模块三：知识问答（`/query`）

三栏追踪布局，让用户看清从问题到答案的完整链路。

#### 3.1 查询配置面板（左栏）
- 知识库选择器（下拉，展示文档数）
- 问题输入框（多行，Shift+Enter 换行，Enter 提交）
- 检索参数：
  - `top_k`：滑块 1-20，默认 8
  - `min_score`：滑块 0-1，默认 0.3
  - `use_llm_check`：开关（标注：启用后重排更精准，耗时增加约2s）
  - `use_llm_score`：开关（标注：启用后评分更可靠，额外消耗 token）
- 历史问题列表（最近10条，点击复用）
- 提交按钮（加载中状态）

#### 3.2 召回 & 重排面板（中栏）

召回通道区：
- 4个通道卡片：稠密召回（pgvector）/ 稀疏召回（BM25）/ 结构化召回 / RRF 融合
- 每个通道展示：候选数量、通道颜色标识
- 关联扩展：扩展了 N 个关联块（图片/表格）

候选列表（重排后）：
- 每条候选：排名序号、来源文件、页码、layer 标签、RRF 分数、重排分数
- 高亮最佳子块（best_child_id）
- 点击候选 → 右侧展开该切块全文
- 上下文窗口预览：展示最终送入 LLM 的上下文文本（可折叠）

#### 3.3 答案 & 评分面板（右栏）

答案区：
- 答案正文（支持 Markdown 渲染）
- 引用编号内联高亮（`[1]` `[2]` 可点击）
- 「无法回答」状态：大字提示 + 原因说明（上下文不足 / 相关度过低）

引用列表：
- 每条引用：编号、来源文件名、页码、内容摘要
- 点击引用 → 中栏对应候选高亮
- 点击「查看切块」→ 抽屉展开完整切块内容

评分区：
- 相关度评分：数值 + 等级（高/中/低）+ 进度条
- 忠实度评分：数值 + 等级 + 进度条
- LLM 评分（若启用）：数值 + 解释文案

链路追踪时间线（可折叠）：
- 混合检索 → 父子重排 → 答案生成 → 质量评分，每步耗时

**验收标准：**
- AC-QRY-01：选择知识库、输入问题、提交后返回答案
- AC-QRY-02：召回通道各自展示候选数量
- AC-QRY-03：答案中引用编号可点击，高亮对应候选
- AC-QRY-04：「无法回答」状态有明确提示
- AC-QRY-05：评分展示相关度和忠实度，带等级文案
- AC-QRY-06：历史问题可复用

### 模块四：策略评测（`/evaluation`）

#### 4.1 评测集管理
- 创建评测集：名称 + 选择知识库 + 上传问答对（CSV 格式：问题,参考答案）
- 评测集列表：名称、问题数、关联知识库、最近运行时间

#### 4.2 策略对比实验
- 选择同一文档，选择 2-3 种切片策略
- 选择评测集（或手动输入测试问题）
- 启动对比实验 → 并行跑多个策略的 RAG 查询
- 结果对比表格：行为测试问题，列为各策略的答案/相关度分/忠实度分
- 支持按分数排序

#### 4.3 评分总览
- 各策略平均分对比（横条图）
- 相关度 vs 忠实度散点图（每个点是一次问答）
- 「无法回答」率对比

#### 4.4 失败样本诊断
- 低分样本列表（分数 < 0.5）
- 点击样本 → 抽屉展开：原问题、答案、引用、评分、召回候选
- 支持「在问答页重新调试」跳转（携带问题和参数）

**验收标准：**
- AC-EVAL-01：可创建评测集，支持 CSV 上传
- AC-EVAL-02：可选择多种策略并行对比
- AC-EVAL-03：对比结果表格支持排序
- AC-EVAL-04：低分样本可跳转到问答页调试

## 前端技术方案

| 项目 | 选择 | 理由 |
|------|------|------|
| 路由分组 | `(rag-api)` | 用户要求 |
| 实时推送 | SSE（`EventSource`） | 单向推送，FastAPI 原生支持，比 WS 轻 |
| 状态管理 | Zustand | 轻量，适合问答页多面板联动 |
| 图表 | Recharts | 已在项目中，保持一致 |
| 文件上传 | 原生 `FormData` + 进度事件 | 无需额外依赖 |
| Mock 模式 | 保留 `NEXT_PUBLIC_DATA_MODE=mock` | 开发调试用 |

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| SSE 在某些代理/CDN 下被截断 | 实时进度失效 | 降级为5s轮询，前端自动切换 |
| MinerU 云端解析耗时长（1-5分钟） | 用户等待体验差 | 任务队列 + 后台处理，用户可离开页面 |
| pgvector 未启动时建库失败 | 核心功能不可用 | 配置页提供连接检测，失败时给出明确提示 |
| mock 模式与真实 API 契约不一致 | 联调困难 | mock 数据结构与 API 响应结构严格一致 |
