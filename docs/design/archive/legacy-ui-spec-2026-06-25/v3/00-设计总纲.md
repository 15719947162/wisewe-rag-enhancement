# WiseWe AI 知识库平台 UI 交互规范 v3.0

**版本：** 3.0  
**创建时间：** 2026-05-29  
**设计风格：** Linear 风格 × Data-Dense Dashboard × 简洁大方  
**核心原则：** 克制、高密度、可信、链路透明

---

## 一、设计语言定义

### 1.1 Linear 风格核心特征（借鉴）

- **极简导航**：左侧窄导航栏，图标+文字，激活态用品牌色填充，无多余装饰
- **卡片设计**：轻边框、无重阴影、圆角 12px，悬停时边框加深而非浮起
- **排版**：大量留白，标题层级清晰，数字用等宽字体，内容比容器更重要
- **交互**：快速响应，150ms 过渡，无装饰性动画，每个操作有明确反馈
- **色彩**：以中性色为主，品牌色点缀关键节点，状态色语义明确

### 1.2 与现有设计令牌的融合

| 令牌 | 值 | 用途 |
|------|----|------|
| 品牌主色 | `#0F3D5E` | 导航激活、主按钮、关键锚点 |
| 品牌主色悬停 | `#14557F` | 主色表面的悬停态 |
| 强调色 | `#0F9F8C` | 成功状态、健康指标、最佳结果标记 |
| 画布背景 | `#F5F7FA` | 应用底层背景 |
| 面板背景 | `#FFFFFF` | 卡片、抽屉、面板 |
| 浮层背景 | `#FCFDFE` | 吸顶栏、顶栏、浮层 |
| 次级背景 | `#F0F4F8` | 斑马纹、悬停背景、次级区域 |
| 界面字体 | Public Sans | 所有 UI 文字 |
| 数据字体 | IBM Plex Mono | 数值、ID、代码、页码 |

### 1.3 不采用的方向

- 不做 Linear 的纯黑暗色模式（我们是浅色为主的数据控制台）
- 不做偏消费级的轻社交风
- 不做重玻璃拟态或装饰性粒子背景
- 不做"酷炫但不透明"的黑盒 AI 界面

## 四、设计成功标准

当 UI 达到目标时，用户应该能做到：

- 在 **10 秒内**看清系统当前状态
- 在 **30 秒内**定位一次建库失败发生在哪个阶段
- 在 **1 分钟内**解释一次答案为什么成立或为什么无法回答
- 在 **2 分钟内**比较两种切片策略的效果差异
