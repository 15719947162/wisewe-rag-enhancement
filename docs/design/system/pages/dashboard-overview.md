# 页面覆盖：总览仪表盘

> 本文件中的规则仅覆盖 `docs/design/system/MASTER.md` 在总览页面中的部分。

## 页面目标

总览页是整个 RAG 系统的控制总台。它必须在一屏内回答用户四个问题：

- 当前哪里是健康的
- 当前哪里被阻塞
- 最近发生了什么变化
- 下一步应该去哪里处理

## 语言覆盖

- 本页面所有可见 UI 文案必须使用 `简体中文`。
- 指标名称、告警文案、队列标签、趋势描述、动作建议必须为中文。
- 只有不可避免的技术标识或缩写可保留英文，但不能成为页面主导语言。

## 布局覆盖

使用 `指挥台首屏 + 诊断下钻`：

1. 首屏上半区
- 左侧：Command Center hero，说明系统职责和当前操作重点
- 右侧：系统健康 + 当前身份面板

2. 首屏中部
- 4 张高信号指标卡
- 指标必须覆盖知识库、文档、切片和最近任务

3. 首屏下半区
- 左侧：入库生产线
- 右侧：风险与告警

4. 首屏底部
- 最近入库任务表格

## 优先级规则

- 系统健康状态优先于装饰性摘要
- 失败或降级阶段必须排在信息性趋势之上
- 任何无法直接驱动行动的指标，都应降低优先级

## 组件

- 指标卡
- 流水线漏斗
- 告警堆栈
- 队列表格
- 微型趋势图

## 视觉气质覆盖

- 相比深度工作页，允许稍微更有管理视角、更透气
- 首屏上方减少高密度表格占比
- 用更强的卡片层级区分摘要和诊断内容
- 不能回到营销首页语气
