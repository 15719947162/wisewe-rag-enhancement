# 清洗规则与质量过滤文档

## 一、解析后清洗规则（core/cleaner/rules.py）

清洗在 **解析 → 切片** 之间执行，作用于 `ContentBlock` 列表。规则按顺序链式应用，每条规则输出的块作为下一条规则的输入。



### 规则详情

#### 1. 空白/极短块（RemoveEmptyBlocks）
- 移除纯空白文本
- 典型噪音：OCR 产生的单字符乱码、空行块
- 单行长度 < 1 的纯标点符号

#### 2. 短文本块（RemoveShortBlocks）
- 默认阈值：2 字符
- 标题块（`type=title`）豁免，避免误删章节标题
- 典型噪音：孤立的标点、单词碎片


#### 3. 纯标点块（RemovePunctuation）
- 计算非字母数字非空白字符占比
- 超过 80% 则判定为无意义符号块
- 典型噪音：`"————————"` `"※※※※※"` 等装饰线


#### 4. 版权/广告（RemoveCopyrightAds）
匹配正则：
```
Copyright © YYYY
All Rights Reserved
版权所有
未经...许可...不得
https://...（URL）
```
仅对长度 < 200 字的块生效，避免误删包含版权说明的正文段落。

#### 5. 同页重复图片（RemoveDuplicateImages）
- 同一页面（`page_idx` 相同）内出现多张相同图片时，保留第一张，移除后续重复项
- 判断依据：优先比较 `image_path`；无路径时比较 `text` 内容（OCR 说明文字）
- 典型噪音：MinerU 解析时同一图片被重复提取多次
- 非图片块（文本、表格、标题）不受此规则影响

### LLM 清洗（可选，core/cleaner/llm_cleaner.py）

在规则清洗之后执行，调用大模型对每批文本块进行：
1. OCR 错误修正（错别字、乱码）
2. 信息价值判断（keep / discard / fix）

**操作类型：**

| 操作 | 含义 |
|------|------|
| `keep` | 保留原文 |
| `discard` | 丢弃（纯广告、版权声明、无实质信息、无法修复的乱码）|
| `fix` | 修正后保留（OCR 错误、明显错别字）|

**环境变量：**
- `LLM_API_KEY` / `DASHSCOPE_API_KEY`：API 密钥
- `LLM_BASE_URL`：API 地址（默认阿里云 DashScope）
- `LLM_CLEANER_MODEL`：模型名称（默认 `qwen-plus`）

## 二、切片后质量过滤（core/cleaner/quality_gate.py）

质量门控在 **切片 → 向量化** 之间执行，作用于 `Chunk` 列表。表格块和图片块豁免所有规则。

### 规则列表

| # | 规则 | 默认阈值 | 触发条件 |
|---|------|----------|----------|
| 1 | 最小字符数 | `min_chars=10` | 切片内容长度 < min_chars |
| 2 | 最大标点比例 | `max_punct_ratio=0.9` | 标点/符号占比 > max_punct_ratio |
| 3 | LLM 质量评分 | `min_score=0`（不启用）| 评分 < min_score（1-5 分制）|

### 规则详情

#### 1. 最小字符数
- 切片后仍可能产生极短片段（如孤立标题、单行列表项）
- 低于阈值直接丢弃，不进入向量化

#### 2. 最大标点比例
- 计算非字母数字非空白字符占比
- 超过 90% 判定为无意义符号切片
- 比清洗阶段阈值（80%）更宽松，避免重复过滤

#### 3. LLM 质量评分（可选）

**模型类型：** 普通文本生成 LLM（Chat Completions），非向量模型。默认使用 `qwen-plus`，通过 OpenAI 兼容接口调用 DashScope。

**评分标准（1-5 分）：**

| 分数 | 含义 |
|------|------|
| 5 | 内容完整、信息丰富、有明确知识点 |
| 4 | 有一定价值，信息基本完整 |
| 3 | 内容有限，但有一定参考价值 |
| 2 | 信息碎片化，价值较低 |
| 1 | 无实质内容（列表符号、空洞描述、无意义片段）|

**运行机制：**
- 每批 10 个切片，每个切片取前 300 字送评
- LLM 返回 JSON 数组，每项包含 `index`、`score`、`reason`
- API 不可用或未配置 Key 时，所有切片默认给 5 分（跳过评分）
- 表格块和图片块豁免，不参与评分

**两种评分模式（UI 可选）：**

| 模式 | 行为 |
|------|------|
| 仅评分，不过滤（默认） | LLM 打分，分数显示在切片浏览器标题行（`⭐ LLM X/5`），所有切片保留 |
| 启用过滤 | 低于 `min_score` 的切片被丢弃，详情见「质量过滤详情」面板 |

**环境变量：**
- `LLM_API_KEY` / `DASHSCOPE_API_KEY`：API 密钥
- `LLM_BASE_URL`：API 地址（默认 `https://dashscope.aliyuncs.com/compatible-mode/v1`）
- `LLM_CLEANER_MODEL`：模型名称（默认 `qwen-plus`）

### UI 参数配置

在侧边栏「⑤ 质量过滤」区域可调整：
- **最小字符数**：0–100，默认 10
- **最大标点比例**：0.5–1.0，默认 0.9
- **LLM 最低评分**：0（不启用）或 1–5（启用评分过滤）

## 三、切片策略 — 三层切片法（core/chunker/hierarchical.py）

三层切片法专为教材类 PDF 设计，产出三种层级的切片：

### 第一层：父级切片（layer="parent"）

- **触发条件**：遇到 `BlockType.TITLE` 标题块
- **内容**：标题文本（如"第三节 静脉输液"）
- **展示**：`📂` 图标，粗体，无缩进
- **用途**：上下文容器，不直接 embedding

### 第二层：子级切片（layer="child"）

- **文本块**：按句子边界分割，最大 `child_max_chars`（默认 600 字符）
  - 检测到操作流程类关键词（步骤/操作/流程/方法/过程/注意事项）时，上限放宽到 1000 字符
- **表格块**：整表一个子级切片，`is_table_chunk=True`
- **图片块**：图片+图注一起切，`is_image_chunk=True`，保留 `image_path`
- **关联**：`parent_id` 指向所属父级切片
- **标注**：如果有对应的增强切片，标题行显示 `✨已增强`
- **展示**：缩进 1 级

### 第三层：增强切片（layer="enhanced"）

- **触发条件**：`enable_enhanced=True` 且子级切片内容 ≥ 50 字（表格/图片块跳过）
- **内容前缀**：`[LLM增强]` — 区分原文和 LLM 生成内容
- **生成内容**：一句话摘要 + 3 个可能的检索问题
- **关联**：`parent_id` 指向对应的子级切片
- **展示**：`💡` 图标，缩进 2 级
- **Token 统计**：累积到 `strategy.total_tokens`，管道流程条中展示

**LLM 配置（可自定义）：**
- 默认复用环境变量 `LLM_API_KEY` / `LLM_BASE_URL` / `LLM_CLEANER_MODEL`
- UI 中可展开「增强切片 LLM 配置」面板，覆盖 model、base_url、api_key

### 关联关系（related_ids）

切片完成后自动建立双向关联（`core/chunker/linker.py`）：

| 规则 | 触发条件 | 效果 |
|------|----------|------|
| 引用匹配 | 文本中出现"图X-X-X"或"表X-X" | 关联到对应的图片/表格切片 |
| 邻近关联 | 图片/表格切片无引用匹配 | 关联到最近的前一个文本切片（向前搜索 5 位） |
| 同 parent | 同一 parent_id 下 | 所有图片/表格 ↔ 所有文本切片互相关联 |

关联是双向的：A 关联 B，则 B 也关联 A。

### CSV 导出新增字段

| 字段 | 说明 |
|------|------|
| `layer` | parent / child / enhanced |
| `parent_id` | 指向父级或子级的 UUID |
| `enhanced_text` | 增强文本原文（不含 `[LLM增强]` 前缀） |
| `related_ids` | 关联切片 UUID 列表（逗号分隔） |

## 四、管道整体流程

```
PDF
 └─ ① 解析（MinerU）→ ContentBlock 列表（文本块 / 表格块 / 图片块）
 └─ ② 清洗
      ├─ 规则清洗（4条，顺序执行）
      └─ LLM 清洗（可选，OCR修复+价值判断）
 └─ ③ 切片（6种策略）→ Chunk 列表
      ├─ 固定长度 / 段落 / 语义化 / 分隔符 / LLM 智能
      └─ 三层切片法（父级→子级→增强，含关联关系）
 └─ ④ 质量过滤
      ├─ 最小字符数（表格/图片块豁免）
      ├─ 最大标点比例（表格/图片块豁免）
      └─ LLM 评分（可选）
           ├─ 仅评分模式：所有切片保留，分数显示在切片浏览器（⭐ LLM X/5）
           └─ 过滤模式：低于 min_score 的切片被丢弃
 └─ ⑤ 向量化（可选，生成 embedding）
 └─ ⑥ 导出（JSON / CSV，含 layer、parent_id、related_ids、enhanced_text 字段）
```
