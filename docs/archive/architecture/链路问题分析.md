# 清洗与质量过滤链路问题分析

> 历史分析文档：成文时仓库仍使用 `src/*` 目录口径。当前对应实现已迁移到 `core/cleaner/rules.py`、`core/cleaner/llm_cleaner.py`、`core/cleaner/quality_gate.py`。

---

## 一、清洗阶段（②）现状

### 1.1 四条规则清单

| 规则 | 逻辑 | 豁免 |
|------|------|------|
| `RemoveEmptyBlocks` | 移除空白或 < 3 字符的块 | 表格块、图片块 |
| `RemoveShortBlocks` | 移除 < `min_chars` 的块（DEFAULT_RULES 中设为 2） | 表格块、标题块 |
| `RemovePunctuation` | 移除标点占比 > 0.8 的块 | 表格块 |
| `RemoveCopyrightAds` | 正则匹配版权声明和 URL | 无 |

### 1.2 规则清洗的三个问题

**问题1：`RemoveShortBlocks` 在默认配置下实际失效**

`DEFAULT_RULES` 中 `min_chars=2`，而 `RemoveEmptyBlocks` 已经过滤了 < 3 字符的块。两条规则形成完全重叠，`RemoveShortBlocks` 从未生效。

建议：将 `min_chars` 调整为 10～15，才能真正过滤低信息量短块。

**问题2：`RemoveCopyrightAds` 缺少中文教材特有噪声模式**

当前 5 条正则均针对英文版权和 URL，对教材中常见的以下噪声无法识别：

- 页眉页脚（"第三章 局部血液循环障碍"单独成行）
- 章节序号孤立行（"1.1.1"、"（一）"单独成行）
- 页码行（"- 69 -"、"第 69 页"）

这类块会在后续切片时被合并进 parent 层，污染章节语境。

建议补充正则：
```python
r"^第\s*\d+\s*[章节页]",          # 页眉章节行
r"^\-\s*\d+\s*\-$",               # 页码行 "- 69 -"
r"^（[一二三四五六七八九十]）$",    # 孤立序号行
r"^\d+\.\d+(\.\d+)*\s*$",         # 孤立编号行 "1.1.1"
```

**问题3：清洗顺序导致统计失真**

当前执行顺序：`RemoveEmpty → RemoveShort → RemovePunctuation → RemoveCopyrightAds`

版权声明（如 "© 2024 版权所有，未经许可不得转载"）中标点比例较高，会被 `RemovePunctuation` 先行移除，`RemoveCopyrightAds` 的统计数字因此偏低，日志中看不到版权清洗的实际效果。

建议调整为：`RemoveCopyrightAds → RemoveEmpty → RemoveShort → RemovePunctuation`

### 1.3 LLM 清洗的三个静默失败风险

**风险1：批处理 index 对齐的静默失败**

`_process_text_batch` 依赖 LLM 返回的 `index` 字段与 batch 内局部序号对齐。如果 LLM 漏返了某个 index（截断、token 超限），该位置的 block 会静默保留原文，不会有任何警告。

当前代码：
```python
except Exception: pass  # llm_cleaner.py
```
吞掉了所有异常，无法区分 API 超时、JSON 解析错误、还是 index 缺失。

建议：记录异常类型和 batch 范围到日志，至少区分网络错误和解析错误。

**风险2：文本截断导致内容丢失**

```python
texts = [{"index": i, "text": b.text[:500], ...}]  # llm_cleaner.py:187
```

文本超过 500 字时被截断发给 LLM 评判，`action=fix` 返回的 `fixed_text` 是基于截断文本修正的，原文超出 500 字的部分被直接丢弃。对教材正文（常见 800～1500 字）会造成内容丢失而非修正。

建议：`action=fix` 时只修正前 500 字，将修正结果与原文后半段拼接，而非整体替换。

**风险3：图片 OCR 文字被绕过**

非 VL 模式下图片块直接跳过（`image_results[i] = block`），但 `ContentBlock.text` 中可能包含 MinerU 提取的 OCR 说明文字，存在错别字/乱码。文本模式的 LLM 清洗完全不处理这部分，OCR 错误会一路传递到切片和向量化阶段。

建议：文本模式下对图片块的 `text` 字段单独走一次规则清洗。

## 二、质量过滤阶段（④）现状

### 2.1 当前质量门控逻辑

```
切片列表
  ├─ 表格块/图片块 → 直接保留（豁免所有规则）
  └─ 文本块
       ├─ 标点占比 > max_punct_ratio(0.9) → 丢弃
       └─ min_score > 0 时 → LLM 评分 1~5
            ├─ score_only=True：全部保留，分数写入 QualityGateResult.scores
            └─ score_only=False：< min_score 的切片丢弃
```

### 2.2 与清洗阶段的重复工作

清洗阶段 `RemovePunctuation` 阈值 **0.8**，质量门控 `max_punct_ratio` 默认 **0.9**，两者都在过滤高标点比例的文本，参数还不一致。

经过清洗（阈值 0.8）的 blocks 切片后，质量门控的 0.9 规则几乎永远不会触发——除非多个块合并后引入新的标点密集区。

建议：质量门控移除标点规则，专注于 LLM 评分，两个阶段职责分离：
- **清洗阶段**：针对原始 block 的格式噪声过滤
- **质量门控**：针对切片后 chunk 的信息价值过滤

### 2.3 三层切片特有的四个质量问题

**问题1：enhanced 层被重复 LLM 评分（浪费 token）**

enhanced 层内容本身就是 LLM 生成的（`[LLM增强] ...`），不可能出现乱码或无意义片段，对它再做 LLM 质量评分是双重付费。

建议：在质量门控入口处豁免 `layer == "enhanced"` 的切片，与表格块/图片块同等处理。

**问题2：related_ids 悬空引用**

质量门控丢弃某个 chunk 后，其他 chunk 的 `related_ids` 中可能仍然保留了被丢弃 chunk 的 ID，形成悬空引用。在向量数据库和知识图谱中查询 related_ids 时会返回空结果，影响关联召回。

建议在质量门控完成后增加一步清理：
```python
valid_ids = {c.id for c in kept_chunks}
for chunk in kept_chunks:
    chunk.related_ids = [rid for rid in chunk.related_ids if rid in valid_ids]
```

**问题3：LLM 评分结果无法持久化**

`QualityGateResult.scores` 是 `dict[int, int]`（chunk_index → score），但 `Chunk` 模型没有 `score` 字段，评分结果无法写入 CSV 和向量库。切片浏览器中计划显示的"⭐ LLM X/5"依赖此字段。

建议在 `src/models/content_block.py` 的 `Chunk` 模型中增加：
```python
llm_score: Optional[int] = None  # 质量门控 LLM 评分 1~5
```

**问题4：parent 层缺少字符数上限检查**

parent 层是多个 child 块合并的章节级内容，字符数可能远超 child 层。当前质量门控只有下限过滤，没有上限检查。过长的 parent 切片（> 2000 字）送入 Re-ranker 时效率下降，也会影响 LLM 生成答案时的注意力分布。

建议：对 `layer == "parent"` 且 `char_count > 2000` 的切片记录警告，后续可考虑自动拆分。

### 2.4 LLM 评分 Prompt 的领域针对性问题

当前评分标准对教材内容存在典型误判：

| 内容类型 | 实际价值 | 当前评分风险 |
|---------|---------|------------|
| 检查项目枚举列表（①②③） | 高——学生常考内容 | 可能被判为"列表符号，价值低" → 2分 |
| 操作步骤单步描述（"3. 接种和孵育"） | 中——需要上下文 | 可能被判为"信息碎片化" → 2分 |
| 诊断标准条目 | 高——临床核心知识 | 形式简短，可能被低估 |

建议在 `_llm_score_chunks` 的 system_prompt 中增加领域说明：
```
教材中的操作步骤、检查项目枚举、诊断标准列举，即使形式简短，
也具有较高的教学价值，请适当给予 3 分以上。
```

## 三、职责划分建议

```
② 清洗（block 级，切片之前）
   职责：去除格式噪声，不判断信息价值
   ├─ 规则清洗：空白 / 版权 / 页眉页脚 / 纯标点
   │   执行顺序：RemoveCopyrightAds → RemoveEmpty → RemoveShort → RemovePunctuation
   └─ LLM 清洗（可选）：OCR 修复 + 图片描述生成

④ 质量过滤（chunk 级，切片之后）
   职责：判断知识价值，不重复格式过滤
   ├─ 豁免：表格块、图片块、enhanced 层
   ├─ 上限检查：parent 层 char_count > 2000 → 警告
   ├─ LLM 评分（可选，领域针对性 prompt）
   └─ 后处理：清理 related_ids 悬空引用 + score 写入 Chunk.llm_score
```

核心原则：清洗阶段不关心内容是否有价值，只关心格式是否干净；质量过滤不关心格式，只关心内容对 RAG 检索是否有用。标点规则不应在两个阶段重复出现。
