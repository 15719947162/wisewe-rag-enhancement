# 04 页面模板规范

## 1. 总览页

### 目标

回答“系统现在是否健康，以及我应该先关注什么”。

### 结构

- 顶部：关键指标
- 中部：离线 / 在线链路健康
- 下部：待处理任务与异常列表

## 2. 知识库页

### 目标

回答“有哪些知识库、包含哪些文档、状态和规模如何”。

### 结构

- KB 列表
- 文档列表
- 文档详情
- chunk 抽屉

### 核心展示

- kb_id
- 文档数
- chunk 数
- 最近更新时间
- 重复文件状态

## 3. 离线建库页

### 目标

回答“一份 PDF 从上传到入库过程中发生了什么”。

### 结构

- 任务头部
- 阶段 stepper
- 阶段工作区
- 结果明细
- 详情抽屉

### 关键视图

- 解析视图
- 清洗视图
- 切片视图
- 质量过滤视图
- 导出/入库视图

## 4. 在线问答页

### 目标

回答“这个问题系统是如何召回、重排、生成与评分的”。

### 结构

- 左：输入与参数
- 中：召回、重排、context
- 右：答案、引用、评分

### 关键视图

- Query Config
- Recall Channels
- Candidate List
- Rerank Results
- Context Window
- Final Answer
- Citation List
- Score Summary

## 5. 评测分析页

### 目标

回答“系统整体效果如何，问题集中在哪一类”。

### 结构

- 总体分数卡
- 趋势图
- 失败样本列表
- 分类诊断
- 单样本详情

### 关键指标

- 平均 relevance
- 平均 faithfulness
- llm_score 分布
- cannot_answer 比例
- 低分样本数

## 6. 系统配置页

### 目标

回答“系统当前用什么模型、参数、库连接与规则”。

### 结构

- 模型配置
- 解析与清洗配置
- 切片策略配置
- 检索与重排配置
- 数据库配置

## 7. 页面模板共性规范

- 页头必须有标题、简述、当前上下文
- 核心操作放在页头右侧
- 列表区与详情区联动
- 重要状态和异常必须首屏可见
