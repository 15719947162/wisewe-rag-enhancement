# 05 图表与数据表达规范

## 1. 原则

- 只在趋势、对比、分布、流转关系重要时使用图表
- 如果用户需要逐项检查、筛选、跳转，优先用表格
- 图表必须服务于分析，不做装饰

## 2. 推荐图表类型

### 2.1 漏斗图

用于：

- PDF -> block -> chunk -> filtered chunk -> embedded chunk
- query -> retrieved -> reranked -> context -> cited

### 2.2 堆叠柱状图

用于：

- 不同切片策略的 chunk 类型构成
- 不同 KB 的文档与 chunk 规模对比

### 2.3 折线图

用于：

- 任务耗时趋势
- 评分趋势
- 每日问答量

### 2.4 散点图

用于：

- relevance vs faithfulness
- chunk 数量 vs 生成质量

### 2.5 热力图

用于：

- 不同 KB / 策略 / 模型组合的评分热区

### 2.6 桑基图

用于：

- 检索 -> 重排 -> 最终上下文 的流转关系

### 2.7 箱线图

用于：

- 不同策略下评分分布比较

## 3. 图表视觉规范

- 网格线要轻
- 标签优先易读
- 控制颜色数量
- 同类业务始终使用同色

## 4. 数据表替代规则

以下场景优先表格，不优先图表：

- citation 明细
- chunk 明细
- candidate 明细
- error log
- 失败样本列表

## 5. 指标表达规则

### 分数

- 统一保留两位小数
- 必须带解释文案或等级

### 比例

- 用百分比显示
- 需要时显示分母

### 时延

- 使用毫秒或秒，不混用

### 数量

- 大数可格式化，但保留原值查看方式

## 6. 图表与链路映射

### 离线建库

- 漏斗图
- chunk 策略对比柱状图
- 质量过滤原因分布图

### 在线问答

- 召回通道贡献图
- rerank 分数分布
- context 来源构成图

### 评测分析

- relevance / faithfulness 散点图
- llm_score 分布图
- 失败原因占比图
