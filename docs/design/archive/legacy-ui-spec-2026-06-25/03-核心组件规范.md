# 03 核心组件规范

## 1. Pipeline Stepper

### 用途

表达离线和在线链路的阶段进展。

### 必含信息

- 阶段名称
- 阶段状态
- 输入数
- 输出数
- 耗时
- 错误或降级提示

### 状态

- 待执行
- 进行中
- 成功
- 降级
- 失败

## 2. Metric Card

### 用途

展示总量、质量、时延、成功率等关键指标。

### 结构

- 标题
- 主数值
- 趋势或变化
- 辅助解释

### 规范

- 同一行只放同类指标
- 不要把解释文案塞进主数值区

## 3. Task Status Card

### 用途

展示单个任务、文档或流水线实例状态。

### 必含信息

- 任务名 / 文档名
- 当前阶段
- 状态
- 创建时间 / 最近更新时间
- 主操作按钮

## 4. Evidence Table

### 用途

承载高密度 chunk、candidate、评测样本、错误记录。

### 必备能力

- 排序
- 筛选
- 固定表头
- 批量操作
- 行选中态
- 与右侧抽屉联动

### 不可省略列

视业务不同，至少包括：

- id
- source
- page
- strategy / layer
- score / status

## 5. Compare Panel

### 用途

做前后对比和策略对比。

### 适用场景

- 原文 vs 清洗后
- semantic vs hierarchical
- dense recall vs final rerank
- answer vs citations

### 建议形式

- 双栏并排
- 三栏并排
- 差异高亮

## 6. Citation Card

### 用途

承载单条引用证据。

### 信息项

- 引用编号
- source
- page
- snippet
- chunk id

### 交互

- 点击跳转对应 chunk
- 支持 hover 高亮答案中的引用编号

## 7. Score Badge

### 用途

统一展示 relevance、faithfulness、llm_score。

### 形式建议

- 数值 + 等级文案
- 色彩不能单独表达高低

例如：

- `0.91 / 高`
- `0.42 / 低`

## 8. Trace Timeline

### 用途

表达一次链路从输入到结果的完整轨迹。

### 节点示例

- query received
- dense recall
- sparse recall
- RRF merge
- rerank
- context window
- answer generated
- citations parsed
- score completed

## 9. Parameter Drawer

### 用途

承载高级参数，避免主界面过载。

### 适合收纳

- 模型配置
- top_k
- min_score
- use_llm_check
- use_llm_score
- chunk size
- enhanced prompt

## 10. Chunk Tree Viewer

### 用途

展示 hierarchical chunking 的 parent / child / enhanced 三层结构。

### 视觉要求

- 层级缩进明显
- 类型标签明显
- table/image/enhanced 有独立标识

## 11. Empty State

### 原则

空状态不能只写“暂无数据”，应解释：

- 为什么为空
- 下一步做什么

## 12. Alert Stack

### 用途

汇总失败、降级、模型回退、质量过低、文档重复等系统提示。

### 优先级

- 阻塞型错误
- 降级型提示
- 信息型通知
